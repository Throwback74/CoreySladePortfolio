Bands in the City

Check it out on Github!

An early group project built on the front end, this app uses jQuery, MaterializeCSS, and a wide range of APIs to provide the user with information about bands and concerts happening nearby. The landing page utilizes the IPinfo API to gather the geolocation data from the user's IP address and populates a carousel with the top 4-6 concerts in their area using the SeatGeek API. The user can click through on any of the concerts listed to buy tickets to the concert from SeatGeek. If the user is looking for a specific artist they can use the artist search, which will provide them with a short bio (LastFM API), a list of their next 10 concerts populated in a collapsible accordian display, and the location of the currently selected concert on a Google Map. Finally the user is given an automatically created YouTube playlist of the artist's top 10 most popular music videos. This page's concert list is retrieved using the Bandsintown API, with the Google Places API used for the map and the YouTube API used for the playlist creation.

Friend Finder Full-Stack App

Check it out on Github!

This app is a relatively simple proof of concept friend finder/dating app. The user is presented with a basic 10 question survey that they can answer using dropdown menus and selecting 1-5 (Completely False - Completely True). After submitting the survey, their responses are run through an algorithim I designed myself that takes the absolute value of each of the differences between the answers for each question for each user and determines who has the most similar answers. (There is a set of dummy data hardcoded in for the first users to be compared with). Also, there is a hidden easter egg in this app initially left for my instructor and TA's, feel free to check it out by entering your name as "Tanner", "Travis", or "Mark"

Memory Game - Built w/ React!

Check it out on Github!

This is the first app built using React, and it is a simple memory game. Storing an array of 30 images from Nightmare before Christmas, the app displays 12 images at a time, reshuffling the entire array and then displaying new images each time you click. The images are shuffled using the Fisher Yates shuffle algorithim. The goal of the game is to remember each image that you have already clicked and avoid clicking the same image twice. The game tracks your current score and your top score of the current session of games. When you lose the game the images all shake and your current score resets to 0.
